## 🚀 Running Locally

### Option 1 — Just Open the File
```bash
# Clone the repo
git clone https://github.com/yourusername/resumelab.git
cd resumelab

# Open directly in browser
open index.html        # macOS
start index.html       # Windows
xdg-open index.html    # Linux
```

> ⚠️ Some browsers block local resources. If fonts don't load, use Option 2.

### Option 2 — Local HTTP Server (Recommended)

**Using Python:**
```bash
cd resumelab
python3 -m http.server 3000
# Visit http://localhost:3000
```

**Using Node.js:**
```bash
npx serve .
# Visit http://localhost:3000
```

**Using VS Code:**
Install the **Live Server** extension → Right-click `index.html` → Open with Live Server.

## 📝 Embedding in Blogger

### Method 1 — iframe Embed (Easiest)

1. Host `index.html` on GitHub Pages (see above)
2. In Blogger, go to **New Post → HTML View**
3. Paste this code:

```html
<div style="width:100%;height:100vh;border:none;border-radius:12px;overflow:hidden;">
  <iframe
    src="https://yourusername.github.io/resumelab/"
    width="100%"
    height="100%"
    frameborder="0"
    scrolling="auto"
    style="border:none;display:block;">
  </iframe>
</div>
```

### Method 2 — Blogger HTML Widget (Full Embed)

1. Go to **Blogger → Layout → Add a Gadget → HTML/JavaScript**
2. Paste the entire contents of `index.html`
3. Save

> ⚠️ Blogger may strip some tags. Method 1 (iframe) is recommended for reliability.

### Method 3 — Custom Blogger Page

1. Go to **Blogger → Pages → New Page**
2. Switch to **HTML view**
3. Paste the full `index.html` content
4. Publish and link it from your navbar

## ⚙️ How It Works

```
User fills form
     ↓
App State (in-memory JS object)
     ↓
Template Renderer (5 options)
     ↓
Live Preview Panel (real-time)
     ↓
html2pdf.js → A4 PDF Download
```

- **State** is stored in a JavaScript object (session-only, no persistence).
- **Templates** are pure CSS classes applied to the same rendered HTML.
- **Preview** updates on every input event via DOM injection.
- **PDF** is generated by cloning the preview div and passing it to html2pdf.js.

## ⚠️ Known Limitations

1. **PDF fonts** — Google Fonts may not embed perfectly in all PDF viewers; ATS tools read text correctly.
2. **No persistence** — Data clears on tab close (by design). Add `localStorage` for persistence.
3. **Mobile PDF** — PDF download works best on desktop browsers.
4. **Blogger CSS conflicts** — Blogger themes may override some styles when not using iframe embed.
5. **Photo in resume** — Not yet supported; most ATS systems recommend text-only resumes.
